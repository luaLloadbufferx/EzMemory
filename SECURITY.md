# Security Policy

## Supported Versions

**Since this is an ACTIVE project, only the LATEST COMMIT is supported.**

| Version | Supported          |
| ------- | ------------------ |
|  MAIN   | :white_check_mark: |

## Reporting a Vulnerability

EzMem deeply interacts with the Windows APIs and target processes.  
**FOR ISSUES** (crashes, leaks), please, open a **public GitHub issue**.
**FOR SECURITY CONCERNS** (vulnerabilities), please, report it via **GitHub's Private Security Advisories**
To help me quickly find and fix the issue, please add a short description of the issue.

## Scope (What's NOT a security issue)
Because EzMemory is a **memory manipulation library**, the following are **OUT** of scope:
- **Game Bans** (e.g. getting banned by an anti-cheat for using this library)
- **Unsafe usage** (e.g. using the library in unsafe ways that are clearly dangerous)
